# AGENTS.md

All agents must follow these rules before editing code:

1. Read `docs/AGENT_PLAYBOOK.md`.
2. Read `docs/adr/` and preserve existing decision history.
3. If you change behavior, update documentation in the same change set.
4. Keep deployment to GitHub Pages functional.
5. Follow the team workflow conventions in `docs/AGENT_TEAM_PIPELINE.md`.

If a requested change conflicts with existing rules/decisions, create a new ADR that supersedes the old decision and document the migration.

For Cursor-specific invocation patterns and prompt templates, see `docs/CURSOR_AGENT_GUIDE.md`.
